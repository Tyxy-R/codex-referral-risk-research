# ChatGPT SSO with Keycloak SAML 2.0

This guide documents a controlled Keycloak SAML 2.0 setup for a ChatGPT workspace.
It is intended for an authorized workspace administrator using a verified domain.

## Safety boundary

- Use a dedicated research tenant and domain that you control.
- Keep ChatGPT SSO set to optional until a full admin login test succeeds.
- Do not use batch referral, activation, or invite automation for production users.
- Do not commit SAML certificates, admin tokens, user exports, auth files, or CSVs.

## Prerequisites

- ChatGPT workspace plan with SSO support.
- Global Admin access in ChatGPT workspace settings.
- A domain verified in the workspace.
- A reachable Keycloak base URL, for example `https://sso.example.com`.
- Keycloak 25+ or equivalent SAML 2.0 IdP.

## OpenAI SSO wizard values

In ChatGPT workspace settings, open the SSO configuration flow and choose SAML.
Copy the values generated by the wizard into Keycloak:

| OpenAI field | Keycloak field |
|---|---|
| ACS URL / Reply URL | Client `Valid redirect URIs` |
| Entity ID / Audience URI | Client ID |
| NameID format requirement | `email` NameID mapper |
| Required user attribute | `email` mapper |

Do not guess these values. The ACS URL and Entity ID can be workspace-specific.

## Keycloak realm

Create a realm for the research tenant, for example:

```text
Realm: chatgpt-research
Frontend URL: https://sso.example.com
User profile: email required
```

Create test users whose email domain matches the verified ChatGPT workspace domain.
Mark email as verified only if the mailbox and domain are controlled by your team.

## SAML client

Create a SAML client:

```text
Client ID: <OpenAI Entity ID / Audience URI>
Client type: SAML
Name: ChatGPT SSO
Enabled: On
Valid redirect URIs: <OpenAI ACS URL>
Master SAML processing URL: <OpenAI ACS URL>
Force POST binding: On
Sign documents: On
Sign assertions: On
Encrypt assertions: Off
Client signature required: Off, unless OpenAI wizard provides an SP certificate
Name ID format: email
```

Use the exact values shown in the OpenAI SSO wizard. Keep assertion encryption
off unless the wizard explicitly instructs otherwise.

## SAML mappers

Add these mappers to the client:

| Mapper type | Name | Value |
|---|---|---|
| User Property | email | `email` |
| User Property | firstName | `firstName` |
| User Property | lastName | `lastName` |
| User Property | NameID | `email` |

The `email` attribute is the important one for ChatGPT SSO routing.

## Metadata

After saving the client, collect IdP metadata from Keycloak:

```text
https://sso.example.com/realms/chatgpt-research/protocol/saml/descriptor
```

Upload this metadata file or copy its certificate and SSO URL into ChatGPT SSO
settings, depending on what the OpenAI wizard asks for.

## Validation checklist

1. ChatGPT domain is verified.
2. SSO mode is optional, not required.
3. Test user exists in Keycloak and has a verified email using the workspace domain.
4. Keycloak client uses the exact OpenAI ACS URL and Entity ID.
5. SAML assertion contains `NameID=email` and an `email` attribute.
6. Assertion encryption is disabled unless explicitly required.
7. Admin can still log in with a fallback method.
8. A non-admin test user can complete SSO login.
9. Only after success, consider making SSO required.
